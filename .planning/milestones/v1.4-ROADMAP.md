## Milestone Summary

**Key Decisions:**

- Used abstract getters for combatantId/combatantName to allow subclass implementations
- Made CombatantModel abstract since it cannot be instantiated directly
- Added backward-compat aliases for gradual migration (CombatUnit, CombatUnitCard, isCombatUnitCard)
- 'MercCard'/'DictatorCard' className strings are BoardSmith classRegistry keys - cannot be renamed

**Issues Resolved:**

- Inconsistent naming (CombatUnit vs Combatant) - now unified as Combatant*
- Missing documentation for class hierarchy - CLAUDE.md created

**Issues Deferred:**

- None

**Technical Debt Incurred:**

- Backward-compat aliases will need removal in future major version
